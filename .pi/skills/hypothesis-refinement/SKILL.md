# Hypothesis Refinement

Use this skill to revise and strengthen a hypothesis in response to feedback.

## When to use

Use this skill when you already have:
- an initial hypothesis
- critique from `adversarial-review`, `novelty-check`, or both
- a desire to improve rather than discard the idea

## Goal

Produce a revised hypothesis that is:
- clearer
- more precise
- more defensible
- more testable
- less overstated
- better aligned with the literature

## Workflow

1. Read the current hypothesis.
2. Read the relevant critique and novelty findings.
3. Separate issues into:
   - fatal flaws
   - fixable weaknesses
   - scope problems
   - wording/claim problems
4. Decide whether to:
   - narrow the claim
   - change the mechanism
   - adjust the target setting
   - weaken the promise
   - add assumptions explicitly
   - split one hypothesis into multiple narrower ones
5. Produce a revised version.
6. Explain what changed and why.
7. If the idea is not salvageable, say so clearly instead of forcing a weak revision.

## Good practices

- Preserve the strongest core intuition when possible.
- Prefer a narrower true claim over a broader fragile one.
- Make assumptions explicit.
- Make falsification conditions sharper.
- Avoid pretending unresolved issues are fixed if they are only partially addressed.

## Output format

### Revised hypothesis
- **Statement:**
- **Motivation:**
- **Mechanism / intuition:**
- **Key assumptions:**
- **How to falsify:**

### What changed
- Change 1
- Change 2
- Change 3

### Addressed concerns
- Which critique points are now better handled

### Remaining concerns
- What is still weak or uncertain

### Recommendation
- Send back to adversarial review / send to novelty check / proceed to planning / discard
